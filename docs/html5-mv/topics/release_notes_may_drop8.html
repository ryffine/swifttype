Release 6 - May drop #8

New features, enhancements, and Resolved Issues

  • In PDF output, empty table rows now have the same row height as the other content rows in the table (US162).
  • Improved the spacing and display of landscape table footnotes in PDF output (US286).
  • In PDF output, images and titles within a fig element now honor the value of the image align attribute. Figure callouts are always left aligned regardless of the image alignment setting (US296).

    The following markup provides an example of how to center a figure within the text column.

    
      <fig>
       <title>A labeled image</title>
       <image href="z600_Front_Back-Test.svg" align="center"/>
       <ol id="ol_f5n_p2d_d1b">
        <li>DVD drive*</li>
        <li>SD card drive</li>
        <li>Power button</li>
        <li>USB inputs</li>
       </ol>
      </fig>
  • Completed the migration of the DITA-OT to version 2.1 (US303).
  • In PDF output, the msgph element output was changed to now use bold text and append a colon (:) to the tagged content (US304).

    For example, the following DITA XML source:

    <msgph>Sample text</msgph>

    will render as

    Sample text:

  • Resolved an issue in PDF output where figure numbers were not being incremented correctly when the content also contained fig elements without a corresponding title (US315).
  • PDF output now includes numbering for section element titles in addition to topic titles when the -Dge.add.subheader.prefix=yes build parameter is set (US316).
